# Linear Feature Issue Template

Use this template for feature requests and product enhancements.

```markdown
# Title

Short concrete desired outcome.

## Problem / Opportunity

What need, pain, or opportunity this addresses.

## Desired Outcome

What should be true after this exists.

## User / Actor

Who needs this and in what situation.

## Current Behavior

What happens today.

## Proposed Behavior

What should happen instead or additionally.

## Context

- Target team:
- Target project:
- Component tag:
- Product area:
- Related workflow:
- Related repos:
- Constraints:
- Non-goals:

## Evidence / Links

- Customer/user note, screenshot, metric, related issue, PR, doc:
```

## Intake Rules

- Create or update the issue in the `Civora` Linear team.
- Do not turn feature intake into implementation planning.
- Do not invent acceptance criteria.
- If behavior is unclear, ask.
- Ask or propose `Target team`, `Target project`, and `Component tag` before finalizing.
- Add or recommend `feature`.
- Add or recommend `llm-refine` when implementation planning is needed.
