# Linear Bug Issue Template

Use this template for bug issues. These fields match the Linear bug issue template.

```markdown
# Title

Short concrete symptom.

## Problem

What is broken and why it matters.

## Expected Behavior

What should happen.

## Actual Behavior

What happens instead.

## Reproduction Steps

1. Step one.
2. Step two.
3. Observe the problem.

## Context

- Target team:
- Target project:
- Suggested labels:
- Page/area:
- Environment:
- Account/tenant:
- Build/version/branch:
- Feature flags/config:
- Related repos:

## Evidence / Links

- Screenshot/log/trace/link:
```

## Intake Rules

- Create or update the issue in the `Civora` Linear team.
- Do not invent reproduction steps.
- Do not invent expected behavior.
- `Evidence / Links` may be empty.
- Ask or propose `Target team`, `Target project`, and matching `Suggested labels` before finalizing; ask whether to add more tags.
- Add or recommend `bug`.
- Add or recommend `llm-refine` when implementation planning is needed.
